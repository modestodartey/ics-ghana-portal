# Repository Instructions

## Working Principles
- Always inspect the codebase before making changes.
- Plan before editing files.
- Keep the UI responsive for phones and laptops.
- Avoid unnecessary packages and dependencies.
- Keep code modular, readable, and clean.
- Preserve future support for tracking, mobile apps, and smartwatch notifications.
- Explain what files were changed and why after each task.
- Do not implement unrelated features.

## Project Direction
- Treat the project as web first, with future expansion to mobile and smartwatch clients.
- Prioritize notifications before tracking unless requirements explicitly change.
- Keep architecture decisions compatible with future student and device tracking workflows.
- Consider privacy, security, and auditability when designing future tracking features.

## Development Expectations
- Prefer simple and maintainable solutions over premature complexity.
- Reuse shared components, services, and utilities where appropriate.
- Keep business logic separated from presentation logic.
- Document significant decisions that affect future architecture or module boundaries.
- Do not add dependencies, frameworks, or tooling unless they are clearly needed for the current stage.

## Scope Control
- This repository is starting in a planning-first stage.
- Do not begin full feature implementation unless explicitly requested.
- When implementation starts, work in small, reviewable steps aligned with the documented scope.
